🧚 MoA-DepthCLIP 童話故事版

原文標題：Lightweight Prompt-Guided CLIP Adaptation for Monocular Depth Estimation

作者：Reyhaneh Ahani Manghotay, Jie Liang

arXiv：2604.01118

📜 Abstract（摘要）

在機器學習的王國裡，有一位名叫CLIP的偉大巫師。這位巫師擁有一項神奇的能力——他能夠同時理解圖像和文字，就像同時精通魔法與語言一樣。CLIP的學識淵博，通曉世間萬物的語義奧秘，他的眼睛能夠看穿圖片背後的意義。

可是，這位巫師有一個小小的困擾：雖然他善於理解「這是什麼」，卻不太擅長判斷「這有多遠」。就像一個博學的學者，能夠描述一座山的形狀，卻說不清楚山峰離自己有多遠。這就是單目深度估計的難題——只用一隻眼睛（一張圖片）來判斷三維世界的距離。

CLIP的學徒們嘗試了很多方法。有的讓巫師進行長時間的特訓（微調），有的試圖教巫師更精確的幾何知識，但這些方法要么太耗時（需要大量參數訓練），要么效果不佳。

後來，一位名叫MoA-DepthCLIP的少年出現了。他帶來了一個巧妙的解決方案——精靈混合物（MoA）！這些精靈是一群小巧而聰慧的助手，它們被安放在巫師CLIP的魔法塔（Vision Transformer）核心部位。精靈們不會打擾巫師原本的魔法，只是輕輕調整他感知世界的方式。

這位少年還設計了一個聰明的策略：他讓精靈們一起工作，有的負責判斷深度區間（就像把距離分成幾個區間：近、中、遠），有的負責精確測量具體數值。兩者配合，就像有人幫你分區又幫你精確瞄準！

為了讓測量更準確，少年還設計了一套複合咒語（複合損失函數），確保精靈們測量的深度符合幾何規律——近的東西確實看起來近，遠的東西確實看起來遠。

他們在著名的NYU Depth V2試煉場進行測試。這裡是深度估計研究者們公認的試金石。

結果令人驚嘆！精靈們的表現大幅超越過去的成績：

  • 準確率（δ1）：從原來的 39.0% 大幅提升至 74.5%
  • 誤差（RMSE）：從原來的 1.176 降低至 0.520

更令人驚喜的是，這些精靈們需要的訓練參數極少——不需要讓整個巫師塔重新學習，只需要少數精靈的幫助，就能讓CLIP巫師掌握深度估計的魔法！

🧙‍♂️ Introduction（介紹）

CLIP家族的崛起

很久以前，偉大的CLIP法師（Vision-Language Master）誕生了。CLIP法師擁有兩種天賦——他能「看」見圖畫的模樣，也能「聽」懂文字的魔法。透過研讀成千上萬本圖文並茂的魔法書，他學會了將圖畫與文字連結在一起的古老智慧。

CLIP法師的厲害之處在於，他不需要從零開始學習——他只要看過幾張圖畫和幾句描述，就能施展「零樣本」或「少樣本」的法術，瞬間辨認出他從未見過的東西。他甚至能夠找出圖畫中躲藏的物體（物體檢測），或者畫出每個像素的輪廓（語意分割）。他的名聲漸漸傳遍各地，甚至延伸到了三維世界，開始理解點雲的秘密。

然而，CLIP法師心裡始終有個遺憾——他擅長的是「模糊的感覺」，卻不懂精確的「測量」。他能看到「這是一座山」「這是一朵花」，卻說不出「山離我有多遠」。

深度山谷的挑戰

單目深度估測，是深度山谷中最神聖的法術之一。學會它的人，能夠為自動導航的馬車指引方向、幫助機器人伙伴行走自如、為擴增實境施展美麗的幻影。這項法術也是其他高深法術的基礎，比如單目三維物體檢測、從一張圖畫重建三維點雲。

過去，練習這項法術的人分為兩派：

一派是全監督學派，他們擁有極度精確的深度地圖作為教材（比如著名的NYU Depth V2寶典），但這些地圖極為昂貴，需要花費大量時間親手繪製。

另一派則是基礎模型學派，他們像收集各國藏書一樣，搜集了無數來自不同地方的資料，用龐大的法力訓練出最厲害的模型。但這些模型體型太過龐大和笨重，普通人根本無法使用。

這兩派方法雖然厲害，卻都有一個問題——要麼需要太多昂貴的教材，要麼需要太多昂貴的法力。難道沒有兩全其美的辦法嗎？

DepthCLIP的初次嘗試

這時，一位名叫DepthCLIP的年輕法師提出了一個大膽的想法：「為什麼不讓CLIP法師用說話的方式來估測深度呢？」

他把深度估測改成了一道選擇題——不是回答「有多遠」，而是比較「比較近」還是「比較遠」。他讓CLIP法師把圖畫的每個區域對應到一段文字描述，比如「眼前的東西」或「遠方的東西」。

這個方法非常新奇，因為它完全不需要特別訓練！但可惜的是，他用手工寫好的提示語太過粗糙，把連續的深度世界硬生生切成幾個大區塊，結果產出的深度地圖總是模模糊糊，缺乏細節。

後來，其他法師嘗試改進這個方法——有人研究如何寫出更好的提示語，有人發明了會看圖畫自覺調整的「深度分桶」，還有人發展出只需要少少幾張圖片就能快速適應的技巧。

MoA-DepthCLIP的誕生

於是，我們的童話進入了嶄新的篇章。我們決定結合兩種強大的法術：

第一種是適配器混合模組（MoA），這是NLP世界中證明有效的「參數效率調優」法術，但從來沒人把它應用在深度估測這種需要精確幾何的任務上。

第二種是分類回歸混合頭，這在傳統深度估測中是眾所周知的標準架構，但從來沒人把它和現代的CLIP適配技術結合在一起。

我們把這兩種法術融為一體：訓練一個輕巧的MoA模組，只微調ViT骨幹的一點點參數，然後把這些調整過的特徵送入分類回歸混合頭。這種前所未有的結合，讓法術既有效率又有空間感知能力。

我們的模型由一個全域場景上下文向量引導——這個向量來自所有文字提示嵌入的平均值。我們的模型則透過一個結合了分類損失和回歸損失（L1損失與尺度不變損失）的複合咒語來約束。

🪄 Method（方法論）

在CLIP王國的深處，存在一座強大的「視覺城堡」——這就是預先訓練好的CLIP視覺骨幹，它是由一種叫做ViT-B/32的Transformer架構所建造的。這座城堡原本已經非常厲害，能夠辨識世界上各種各樣的圖片，但國王希望它能夠更擅長一種特殊的本領——深度估測（Depth Estimation），也就是「看見事物遠近」的能力。

於是，國王召募了一個特殊的魔法團隊，叫做MoA團隊（Mixture-of-Adapters，混合適配器團隊）。這支團隊不會重新訓練整座城堡（那太耗時又耗力！），而是巧妙地在城堡的幾層關鍵塔樓中安裝「智慧插件」，讓城堡在保留原有能力的同時，獲得新本領。

MoA團隊的三位重要成員

每個MoA模組就像一個小型魔法教室，由三位重要成員組成：

  • 專家們 (Experts)：一群各有所長的小精靈，每人專精一種魔法技能
  • 門控網路 (Gating Network)：一位智慧的「分發隊長」，決定哪個小精靈該出馬
  • 殘差混合 (Residual Mixing)：一條「星光通道」，將小精靈的魔法輸出巧妙地融入城堡主體

專家小精靈（Expert Architecture）

每位「專家小精靈」都是一個兩層的神經網路（MLP），但他們有個特別的「瓶頸身材」——這意味著他們會先把龐大的資訊壓縮成小小的「魔法瓶」，處理完後再恢復成原本的大小。

公式： $$\mathrm{Expert}(x) = W_2\,\sigma(W_1 x)$$

白話解釋：

  • $x$ 是進入小精靈的能量（輸入特徵），維度是 $d$（對於 ViT-B/32，$d = 768$，就像一條有 768 個位置的走廊）
  • $W_1$ 是第一扇「壓縮門」，將 $d$ 維度的能量變成 $d_b$ 維度（$d_b = 64$，就像把走廊擠進只有 64 個位置的房間）
  • $\sigma$ 是 GELU 魔法，一種活化函數，能讓能量「活化」並產生非線性變化
  • $W_2$ 是第二扇「解壓門」，將 $d_b$ 維度變回 $d$ 維度
數值範例：輸入維度 768 → 壓縮到 64 → 活化 → 還原到 768。這樣的設計讓我們只需要訓練兩個小矩陣（$W_1$ 和 $W_2$），而不是整座城堡的所有魔法石！

門控隊長（Gating Network）

這位隊長的工作非常細心——他不是隨便指派一個小精靈，而是對每個Token單獨決定該由哪位專家來處理！

公式： $$g_i = \operatorname{softmax}\!\left(\frac{G(x_i)}{\tau}\right)$$

白話解釋：

  • $x_i$ 是第 $i$ 個 Token 的表示（就像城堡中某個位置的情報）
  • $G$ 是一位「評分官」，使用線性投影將 $d$ 維的 Token 表示投影到 $K$ 維的空間（$K$ 是專家的數量）
  • $\tau$ 是「溫度參數」——控制分發的「軟硬度」：
    • $\tau$ 很大（如 10）→ 分發很平均，大家都有機會
    • $\tau$ 很小（如 0.1）→ 只信任最厲害的那位，其他小精靈沒事做
數值範例：若 $K = 4$，某 Token 可能得到 $[0.1, 0.7, 0.15, 0.05]$，表示由第 2 位專家主責（70% 的權重）。

選擇性放置（Selective Placement）

城堡有那麼多層塔樓，MoA團隊該住在哪裡呢？

研究者們做了很多實驗，發現：不要每層都住！只需要選擇幾個關鍵樓層（例如第 2、5、8、11 層）安裝 MoA 模組就夠了。這樣可以達到「剛剛好」的平衡——既有多少的適應能力，又不會增加太多參數負擔。

童話比喻：就像一支小型顧問團隊，不需要在公司的每個部門都設立辦公室，只需要在那幾個最需要改革的部門扎根即可！

MoA-DepthCLIP來到了深度估計領域最重要的競技場——NYU Depth V2！

📖 Conclusion（結論）

在AI王國裡，有一位名叫CLIP王子的勇者，他天生擁有看懂世界的雙眼，卻總是無法精確測量距離的魔法。CLIP王子雖然見多識廣，卻對「深度」這門藝術一竅不通——他只能看到物體的輪廓，卻說不出眼前的花朵究竟距離他三步還是十步。

這天，一位年輕的魔法師MoA-DepthCLIP挺身而出，決定幫助CLIP王子完成這項不可能的任務。

首先，魔法師召喚了幾位小巧玲瓏的適配器小精靈（Adapter Modules），這些小精靈非常節儉，只需要極少的力量（參數）就能施展魔法。他們悄悄潛入CLIP王子的心臟——ViT-B/32 backbone——在不改變王子本性的情況下，賦予他測量深度的能力。

魔法師還請來了一位全局的語義上下文向導（Semantic Context Vector），這位向導就像童話中的智慧老人，總是溫柔地提醒小精靈們：「記住，這裡是牆壁，那裡是地板喔！」

但故事還沒結束！MoA-DepthCLIP發現，單純用一種方法無法達到完美。於是他創造了一套混合預言術：一部分小精靈負責猜測深度區間（就像預言家說「它在1到2公尺之間」），另一部分則直接算出精確數值。兩種方法攜手合作，就像城堡裡的弓箭手與魔法師並肩作戰。

令人驚喜的事情發生了！原本的DepthCLIP只能達到39.0%的精準度（δ1），就像一個近視的騎士總是射偏靶心。但在MoA-DepthCLIP的魔法加持下，精準度飆升至74.5%——幾乎是原來的兩倍！而測量誤差（RMSE）從原本的1.176大幅下降到0.520，就像是測量師終於戴上了正確的眼鏡。

最令人敬佩的是，這一切驚人的進步，只需要極少的新參數——就像魔法師用區區幾顆魔法粉塵，就完成了偉大的轉變。

這個童話告訴我們：

  1. 🦄 用小法術也能變強：MoA這種輕量級適配器，讓一般硬體也能玩轉強大的VLMs
  2. ⚔️ 結合古法與新招：把經典的混合預測架構（分類+回歸）與現代的VLM適配策略結合
  3. 🌍 語意與幾何可以兼顧：CLIP的語意理解 + 深度估計的幾何精確，兩者不衝突，反而相輔相成！
