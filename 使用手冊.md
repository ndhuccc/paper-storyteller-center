# 論文說書改寫中心 使用手冊（繁體中文）

本手冊針對目前專案版本撰寫，內容涵蓋安裝、啟動、Web 操作流程、CLI 指令、API 端點、資料結構與故障排除。

## 1. 系統簡介

論文說書改寫中心是一個本地優先的論文工作平台，整合以下能力：

- PDF 轉換為說書風格 HTML（背景任務）
- HTML / 論文內容向量索引（LanceDB）
- 語意搜尋
- 論文問答（Q&A）
- 論文管理（刪除、重新命名、編輯顯示名稱）
- 生成任務追蹤（提交、查詢、重試、取消、handoff）

系統同時提供：

- Web 介面（Flask + Alpine.js）
- CLI 介面（paper_center.py）
- REST API（/api/*）

## 2. 核心路徑與檔案

### 2.1 專案目錄（程式碼）

- 目前工作目錄：本專案根目錄
- 主要入口：server.py、start.sh

### 2.2 執行資料目錄（使用者資料）

系統會將主要資料寫入：

- ~/Documents/Storytellers

其中包含：

- HTML 輸出檔：*.html
- 向量資料庫：papers.lance
- 索引中繼資料：.papers_index_meta.json
- 任務紀錄：.jobs/*.json
- 環境變數檔（可選）：.env

### 2.3 上傳暫存目錄

- 專案內 uploads/
- Web 端 PDF 上傳會先放這裡，再提交背景任務
- 內建清理策略：
  - 保留天數：14 天
  - 最多檔案數：200
  - 單檔大小上限：50MB（API 層）

## 3. 先決條件

### 3.1 作業系統

- Linux（目前你的環境即 Linux）

### 3.2 Python 與必要套件

建議 Python 3.10+。至少需具備：

- flask
- flask-cors
- markdown
- lancedb
- pymupdf
- python-dotenv
- google-generativeai
- streamlit（若要使用舊 GUI）

範例安裝：

```bash
python3 -m pip install flask flask-cors markdown lancedb pymupdf python-dotenv google-generativeai streamlit
```

### 3.3 Ollama（本地模型）

需先安裝並啟動 Ollama，並拉取模型：

```bash
ollama pull qwen3-embedding:8b
ollama pull gemma4:e2b
ollama pull deepseek-r1:8b
```

預設 Ollama 位置：

- http://localhost:11434

### 3.4 Gemini API Key（建議）

系統的主要改寫模型與主要 Q&A 模型預設走 Gemini。請至少提供以下其中一個環境變數：

- GOOGLE_API_KEY
- GEMINI_API_KEY

可放在：

- ~/Documents/Storytellers/.env
- ~/.env

範例：

```env
GOOGLE_API_KEY=你的金鑰
```

若未提供 Gemini Key，系統會依 fallback 鏈嘗試其他模型（例如本地 Ollama、MiniMax）。

## 4. 啟動與停止

### 4.1 一鍵啟動（建議）

在專案根目錄：

```bash
./start.sh
```

此腳本會：

1. 切換到專案目錄
2. 啟動 Flask 伺服器（server.py）
3. 預設監聽 0.0.0.0:8501

### 4.2 直接啟動

```bash
python3 server.py
```

可用環境變數調整埠號：

```bash
PORT=8502 python3 server.py
```

### 4.3 服務健康檢查

```bash
curl -s http://127.0.0.1:8501/api/health
```

成功會回傳狀態 ok。

## 5. Web 介面完整操作流程

打開瀏覽器：

- http://127.0.0.1:8501

### 5.1 側欄區塊

1. 論文數量
- 顯示目前 manifest 可見論文數

2. 重建索引
- 重新掃描資料並重建向量索引
- 在你做了批次變更（新增 HTML、手動移動檔案）後，建議執行一次

3. 所有論文
- 顯示論文列表
- 點擊可在 modal 內嵌檢視 HTML
- 可下載 HTML

4. 管理（刪除論文）
- 會同時刪除索引資料與 HTML 檔
- 不可復原

5. 管理（重新命名）
- 變更實體 HTML 檔名
- 會觸發索引重建流程
- paper_id 會隨檔名規則影響（因 stem 參與識別）

6. 管理（編輯顯示名稱）
- 只改顯示名稱，不改檔名與 paper_id
- 會同步更新 metadata（display_titles）
- 目前版本亦會同步嘗試更新 HTML 的 title 與第一個 h1

### 5.2 論文狀態說明

- ready：有 HTML 且已索引，可搜尋與 Q&A
- generated_not_indexed：已有 HTML，但未完成索引
- index_only：只有索引，沒有 HTML
- unavailable：資料不足或未完成

### 5.3 語意搜尋分頁

操作流程：

1. 輸入關鍵字
2. 按搜尋
3. 檢視結果列表（含相似度指標）
4. 可勾選多篇加入 Q&A 範圍
5. 可直接開啟該篇 HTML

建議搜尋語句：

- 方法名（例如 knowledge distillation）
- 任務目標（例如 noise elimination）
- 問題意圖（例如 這篇在解決什麼）

### 5.4 Q&A 分頁

操作流程：

1. 可先從搜尋結果勾選目標論文（縮小回答範圍）
2. 輸入問題並送出
3. 系統顯示回答、引用來源、引擎順序
4. 可點來源快速回到論文

AI 引擎順序：

- 可用上下鍵調整優先順序
- 可還原預設順序
- 執行時會依順序 fallback

快捷鍵：

- 問題輸入框支援 Ctrl+Enter 送出

### 5.5 說書改寫分頁（生成任務）

有兩種輸入模式：

1. PDF 自動掃描
- 拖曳或選檔上傳 PDF
- 選風格與風格參數
- 是否 auto-index
- 提交後進入背景任務

2. 手動輸入改寫單元
- 輸入論文標題
- 新增多個單元（標題 + 內文）
- 可批次貼上（以 === 分隔單元）
- 提交後同樣走背景任務

任務列表（最近 8 筆）：

- 可查看狀態：pending / running / succeeded / failed / canceled
- 可展開細節（模型、錯誤、警告、步驟）
- 失敗可重試
- 執行中可取消
- 成功後可 handoff：
  - 開啟改寫結果
  - 搜尋這篇
  - 詢問這篇

## 7. CLI 使用方式

CLI 入口檔：paper_center.py

```bash
python3 paper_center.py init
python3 paper_center.py rebuild
python3 paper_center.py search "knowledge distillation"
python3 paper_center.py ask "這篇論文的核心貢獻是什麼？"
```

說明：

- init / rebuild：重建索引
- search：語意搜尋
- ask：RAG 問答

## 8. API 使用方式

Base URL：

- http://127.0.0.1:8501/api

常用端點：

- GET /health：健康檢查
- GET /papers：論文列表
- GET /papers/{paper_id}/html：讀取 HTML
- GET /papers/{paper_id}/download：下載 HTML
- DELETE /papers/{paper_id}：刪除論文
- PATCH /papers/{paper_id}/rename：重新命名
- PATCH /papers/{paper_id}/display-name：更新顯示名稱
- POST /search：語意搜尋
- POST /answer：問答
- POST /index/rebuild：重建索引
- GET /jobs：任務列表
- GET /jobs/{job_id}：任務細節
- POST /jobs/{job_id}/retry：重試
- POST /jobs/{job_id}/cancel：取消
- POST /jobs/submit：提交任務（multipart 或 JSON）
- GET /styles：風格列表
- GET /engines：引擎列表

範例：語意搜尋

```bash
curl -s -X POST http://127.0.0.1:8501/api/search \
  -H 'Content-Type: application/json' \
  -d '{"query":"knowledge distillation","top_k":10,"threshold":0.0}'
```

範例：Q&A

```bash
curl -s -X POST http://127.0.0.1:8501/api/answer \
  -H 'Content-Type: application/json' \
  -d '{"question":"請比較兩篇方法的差異"}'
```

範例：更新顯示名稱

```bash
curl -s -X PATCH http://127.0.0.1:8501/api/papers/<paper_id>/display-name \
  -H 'Content-Type: application/json' \
  -d '{"display_name":"新的顯示名稱"}'
```

## 9. 維運與資料管理

### 9.1 建議備份項目

至少定期備份以下路徑：

- ~/Documents/Storytellers/*.html
- ~/Documents/Storytellers/papers.lance
- ~/Documents/Storytellers/.papers_index_meta.json
- ~/Documents/Storytellers/.jobs/

### 9.2 清理與容量管理

- uploads/ 為暫存，不建議長期依賴
- 若磁碟壓力高，可定期清理舊任務 JSON 或舊 HTML

### 9.3 Runtime 選擇

auto-index 會檢查目前 Python 是否具備 lancedb。
若目前 Python 不可用，可透過環境變數指定：

- PAPER_STORYTELLER_PYTHON
- STORYTELLER_PYTHON

範例：

```bash
export PAPER_STORYTELLER_PYTHON=/home/linuxbrew/.linuxbrew/bin/python3
```

## 10. 常見問題與排除

### Q1. 啟動後無法開啟頁面

檢查：

1. 是否成功啟動 server.py
2. 埠號是否被占用
3. 是否防火牆阻擋

可嘗試：

```bash
PORT=8502 python3 server.py
```

### Q2. 問答或改寫失敗，出現金鑰錯誤

若訊息含 missing GOOGLE_API_KEY/GEMINI_API_KEY：

1. 設定 GOOGLE_API_KEY 或 GEMINI_API_KEY
2. 重新啟動服務

### Q3. 問答或改寫很慢 / 失敗

檢查：

1. Ollama 是否運作中
2. 本地模型是否已 pull 完成
3. 系統資源（CPU/RAM）是否不足

### Q4. 生成成功但搜不到

可能是 generated_not_indexed 狀態。請：

1. 勾選 auto-index 再提交新任務，或
2. 手動點擊重建索引

### Q5. 刪除後仍看到舊資料

請執行重建索引，並重新整理頁面。必要時重啟服務。

### Q6. start.sh 出現 Exit Code 137

137 常見原因是程序被系統終止（例如記憶體不足）。
建議：

1. 先關閉其他大型程序
2. 降低同時任務數
3. 監看系統記憶體與 swap

## 11. 安全與操作建議

- 不要把 API key 寫進版本控制
- 共享環境務必更換 FLASK_SECRET_KEY
- 對外開放服務時建議加上反向代理與存取控管
- 重要資料先備份再做大量刪除或改名操作

1. 安裝 Python 套件與 Ollama 模型
2. 設定 GOOGLE_API_KEY（建議）
3. 啟動服務：./start.sh
4. 開啟 http://127.0.0.1:8501
5. 在說書改寫頁上傳一份 PDF
